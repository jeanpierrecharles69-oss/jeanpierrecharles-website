§4 Graceful degradation Supabase — preuve par lecture

Scénario : SUPABASE_URL ou SUPABASE_SERVICE_ROLE_KEY absente des env vars Vercel.

// api/_lib/supabase.ts (v1.0.0)
export const supabase: SupabaseClient | null =
    SUPABASE_URL && SUPABASE_SERVICE_ROLE_KEY
        ? createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, { ... })
        : null;

Appelants :

  • diagnostic-request.ts : if (supabase) { ... insert } else { logSupabaseUnavailable('diagnostic-request'); } — pas de blocage.
  • mollie-webhook.ts : if (supabase && metadata.request_id) { ... update } — webhook continue sans update, emails envoyés quand même (C3-bis préservé).
  • diagnostic-status.ts : if (!SUPABASE_ENABLED || !supabase) return res.status(503) — endpoint répond proprement 503 au client PowerShell, pas de crash process.

Conséquence : le site fonctionne identiquement à v3.4.6 tant que JP n'a pas activé Supabase. Migration zero-downtime.
